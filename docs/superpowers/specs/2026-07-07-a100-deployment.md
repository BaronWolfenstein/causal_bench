# A100 Deployment Spec ‚Äî causal_bench GPU work

**Status:** living document. Covers what's built today (SMC/IPCW core, localization diagnostic). The **diffuse_directly** and **QLoRA** sections are stubs to be filled when that work lands (¬ß6).

**Target hardware:** on-prem **8√ó A100 40GB** Lambda box, reached over a Tailscale tunnel (not metered cloud ‚Äî constraints are physical/operational, not billing). See the localization GPU build spec for the encoder-side detail; this spec is the deployment/porting view across the whole repo.

**Core principle carried from all prior design:** stay at the **collective / library layer**. The multi-GPU barrier is an NCCL collective; NCCL owns device-side sync, stream ordering, and memory fencing. Do **not** hand-write CUDA kernels, cooperative-groups grid syncs, or CPU memory fences (membarrier) ‚Äî every one of those is below the boundary and means reimplementing NCCL/CUB. Systematic resampling is `cupy.cumsum` + `cupy.searchsorted` (CUB `DeviceScan`), already optimal and self-synchronizing.

---

## 0. Environment & dependencies

- **Torch (CUDA 12 build)** + **`cupy-cuda12x`** ‚Äî add under a `[gpu]` extra in `pyproject.toml`; never import at module top level (lazy-import inside functions so CPU-only installs and CI stay torch-free). The sampling package already lazy-imports cupy in `backend.array_namespace`.
- **Localization encoder stack:** `femr`, `transformers`, `torch` ‚Äî **version-coupled**; pin to the MOTOR/CLMBR model-card README exactly at load time.
- **Device resolution:** `resolve_device("auto")` (cuda ‚Üí mps ‚Üí cpu) for single-device torch; `cuda_available_devices()` (reads `CUDA_VISIBLE_DEVICES`, falls back to `torch.cuda.device_count()`, returns `[]` on a CPU box) gates the multi-GPU path.
- **Shared-box discipline** (the box is likely multi-user ‚Äî "8√óA100" may be "3 free right now"):
  - `nvidia-smi` before launching; note idle GPUs.
  - Pin with `CUDA_VISIBLE_DEVICES=<free, NVLink-adjacent ids>` **before** `torchrun`; `--nproc_per_node` takes the first N *visible* devices.
  - `nvidia-smi topo -m` to confirm NVLink adjacency (`NV#` = NVLink; `PHB`/`PXB`/`SYS` = PCIe). On an NVSwitch box every pair should read NVLink.
  - Every long job inside `tmux`/`screen` ‚Äî a Tailscale reconnect or closed lid must not kill a multi-hour run.
  - **Tailscale is the control plane, not the data path** ‚Äî land/generate data on the box's local disk; do not push large MEDS/embedding parquet across the tunnel.
- **Storage:** run `df -h` first. Footprint is small (embeddings parquet tens of MB‚ÄìGB; particle state is MB; QLoRA adapters MBs). Internal NVMe is the hot-read path. An external TB5 SSD is **not needed** at these sizes ‚Äî only helps cold snapshots/dataset-at-rest if the box is genuinely out of space, and even then keep hot reads on internal NVMe.

## 1. SMC/IPCW core ‚Üí multi-GPU (the deferred sub-plan, made concrete)

Today: `causal_bench/sampling/` is numpy-first; `run_smc(device="cpu")` is the validated reference. `device="cuda"` is a **scaffolded seam only** ‚Äî the hot loop still calls `np.*`. Deployment work, in order:

### 1a. Port the hot loop to the array namespace `xp`
The one real code gap flagged in review. Route through `xp = array_namespace(device)`:
- `smc_step`: the resample branch's `np.zeros(len(idx))` ‚Üí `xp.zeros`; `state.particles[idx]` gather works on cupy.
- `weights.normalize_log_weights`: `np.max`/`np.exp`/`np.log` ‚Üí `xp.*` (keep the all-`-inf` raise ‚Äî it now surfaces total weight collapse loudly).
- `resample.systematic_resample`: `np.cumsum`/`np.searchsorted` ‚Üí `xp.*` (cupy backs these with CUB ‚Äî do not reimplement).
- Keep `run_smc` returning **host numpy** via `to_numpy(...)` (already done) so callers/estimators stay device-agnostic.
- Acceptance: `run_smc(device="cuda")` numerically matches `run_smc(device="cpu")` on the same seed to tolerance.

### 1b. Collectives ‚Äî the barrier
- **`all_reduce(Œ£w, Œ£w¬≤)`** across ranks ‚Üí Kish ESS (the only true sync point). Fuse the **weight-finiteness check** into this barrier (cheap; catches the all-`-inf` collapse at the reduce).
- **`all_gather`** weights so every rank holds `w[1..N]`.
- **`all_to_all`** particle redistribution (a survivor on rank 3 may be ancestor of particles owned by rank 0).
- Via `torch.distributed` (NCCL). No custom kernels.

### 1c. Gating & topology
- `cuda_available_devices()` ‚Üí if empty, **fall back to single-process CPU `run_smc`** (never crash on a CPU box). If non-empty, init the process group over exactly those ids, `nproc_per_node = len(ids)`.
- Pin NVLink-adjacent ids; set `CUDA_VISIBLE_DEVICES` before `torchrun`.

### 1d. Minimize data movement (roofline: the gather is memory-BW-bound, ‚âà0 arithmetic intensity)
- **Island / local resampling** as the default: each rank resamples its own sub-population, occasional global exchange only ‚Äî removes the `all_to_all` at a small bias/variance cost.
- **Ancestor-index indirection:** keep the int32 lineage vector (already produced), defer the physical state gather until states are next written ‚Äî avoids redundant copies when one survivor is duplicated massively (the rare-event degeneracy regime).

### 1e. Correctness & validation on-box
- **Shared-seed systematic resampling ‚Üí byte-identical indices across ranks.** The numpy oracle (`sharded.sharded_systematic_resample`, Task 6) is what the real `all_to_all` must match.
- **Decisive test:** distributed run == single-GPU reference SMC on the same seed, to numerical tolerance. Validate at **2 ranks** first (enough to exercise `all_reduce` + `all_to_all`).
- Then the **throughput sweep** at 2/4/8 ranks (free on your own box; was previously out of scope).
- Post-resample weights normalize to uniform on every rank (assert).

## 2. Localization diagnostic ‚Üí real encoders (deployment view)

The decision procedure (`run_diagnostic`, Tests A/B/B‚Ä≤/B‚Ä≥/C) is **encoder-agnostic numpy ‚Äî it runs unchanged on any machine.** Only the *production of its input arrays* is GPU work:

- **Frozen encoder:** MOTOR (`StanfordShahLab/motor-t-base`, primary) or CLMBR-T-base (fallback) via FEMR; `eval()`, `torch.no_grad()`, no gradients; pinned FEMR/transformers/torch versions. Research stand-in for SMB's JEPA ‚Äî non-commercial license, diagnostic only, do not ship.
- **MEDS cohort:** extract-once ‚Üí persist embeddings to parquet; all downstream iteration is GPU-free.
- **`E_eval` decoupled encoder (#88):** a **second** frozen encoder (e.g. MOTOR generates, CLMBR evaluates). The full Tier-2 metric-hacking guard needs generated samples rendered to MEDS then re-encoded by `E_eval` ‚Äî that render step is ELF discretization and lands with **diffuse_directly** (¬ß6). Tier-1 (separation AUC of the real reference sets in `E_eval`) is available immediately.
- **Recon / guided arrays** (`recon_b`, `rare_guided`, ‚Ä¶) are produced by the diffusion (¬ß6) and fed to `run_diagnostic`, which then returns the terminal.

## 3. Profiling on-box ‚Äî what only the box can tell us

**Measure on the A100 (uninformative off-box):**
- Absolute wall-clock; **all-to-all comm cost** (O(N¬∑dim) over NVLink); memory-bandwidth / batch-throughput tradeoffs.

**Already CPU-validated ‚Äî do NOT re-derive on-box:**
- Resample-trigger rate (property of how far R sits from base mass), per-particle scaling (O(N), not O(N¬≤)), ESS/weight-degeneracy numerical health, and the distributed==serial index invariant. These are hardware-independent and settled.

**Roofline split to hold in mind:** score net (¬ß6) ‚Üí *maximize FLOPs* (bf16/Tensor Cores, large batch, `torch.compile`); resampling gather ‚Üí *minimize movement* (island / index-indirection). Opposite sides of the roofline.

---

## 4. Checkpointing

- **SMC has no trainable weights** ‚Üí no weight/optimizer checkpoints. Only persist embeddings/particle state to parquet for long runs / resume.
- **Fault tolerance:** for any long GPU job, snapshot enough to resume after crash/preemption (shared box). For SMC that's the particle ensemble + RNG state + step index.
- (Training checkpoints ‚Äî weights + Adam state, resume/rollback ‚Äî belong to **diffuse_directly** ¬ß6. Note: *activation/"gradient" checkpointing* is a VRAM technique, not disk. Checkpoints are needed even when the model fits in VRAM ‚Äî for crash recovery and model selection, not memory. At our sizes they're small; internal NVMe suffices, TB5 SSD unnecessary.)

---

## 5. Honest boundaries / non-goals

- **No custom CUDA kernels, cooperative groups, grid/warp/block sync, or CPU memory fences (membarrier).** Collective layer only; cupy/CUB for scan/searchsorted.
- **`device="cuda"` for SMC is not yet end-to-end validated** ‚Äî ¬ß1a is the prerequisite before any cuda claim.
- The localization decision procedure is not GPU work and must not be "ported" ‚Äî it stays CPU/numpy.
